# 法律场景 MVP — AI 相关技术学习路径

> 目标：能**独立实现**「本地文档检索 + 可选 RAG 问答 + 安全边界」；知道学到哪一层就够做 MVP，不必一口气读完所有论文。

---

## 1. 学到什么程度（分角色简化）

| 阶段 | 你要能做什么 | 对应 MVP |
|------|----------------|----------|
| **L1 基础** | 会调嵌入模型 API 或本地嵌入；会算相似度；理解 token 与上下文窗口 | 关键词检索升级版、简单语义检索 |
| **L2 RAG** | 会切分文档、建向量库、检索 Top-K、拼 prompt、要求带引用 | 法条/合同片段问答（仅基于用户文档） |
| **L3 产品化** | 会评估幻觉、做拒答、做引用校验、做脱敏与离线开关 | 可上线的「助手」形态 |
| **L4 进阶（可选）** | 了解最新长上下文、Agent、法律 NLP 评测 | 二期差异化，**非 MVP 必须** |

**MVP 建议：至少稳在 L2，向 L3 靠齐（引用与免责声明）。**

## 2. 数学基础：要不要、要多少

做你这个 **RAG + 检索 + 调 API** 的 MVP，**不需要**先啃完一本高数/线代教材；但需要知道**几件事在算什么**，否则看 embedding、相似度、评测指标会懵。

### 2.1 MVP 够用档（优先掌握）

| 主题 | 用到哪里 | 学到什么程度 |
|------|-----------|----------------|
| **向量与维度** | 一条文本 → 一个 embedding 向量（如 768 维） | 知道「向量 = 一串数」「维度 = 长度」即可 |
| **点积 / 余弦相似度** | 向量检索：谁和用户问题「更像」 | 会**直觉**：同向更相似、夹角小更相似；能读 `cosine similarity` 公式即可，不必手推证明 |
| **均值、比例、Top-K** | 看检索命中率、简单 A/B | 中学统计够用 |
| **softmax（直觉）** | 读论文里 attention 示意图 | 知道「把一组数变成和为 1 的概率分布」即可，**不必会手算** |
| **对数与 log 概率（可选）** | 读 LLM loss、perplexity | 知道 log 单调、用来把连乘变加法即可 |

### 2.2 建议了解、但可延后

| 主题 | 何时需要 |
|------|-----------|
| **线性代数：矩阵乘、特征空间（浅层）** | 你想读懂 Transformer「QKV」在干什么时；CS224N 注意力那几课会带 |
| **概率基础：条件概率、贝叶斯直觉** | 做不确定性、校准、评测指标扩展时 |
| **微分（极浅）** | 只有你要**自己训练或微调模型**时才必须；**MVP 只推理/调 API 可跳过** |

### 2.3 和「工程实现」的对应关系

- **只调嵌入 API + 向量库**：实现上就是「算相似度 / 用库自带的 distance」→ **2.1 足够**。  
- **自训分类器、改模型结构**：才需要把 **§2.2（线代、概率、微分等）** 系统补起来。

### 2.4 补数学的轻量资源（任选其一打底）

- **3Blue1Brown** — *Essence of Linear Algebra* / *Neural Networks*（YouTube，建立几何直觉）  
- **Khan Academy** — 向量、点积、余弦（短模块）  
- **Andrew Ng — Machine Learning Specialization**（Coursera）前几周的向量、梯度**概念课**（不必做完整个 specialization）  

结论：**需要数学基础，但门槛是「检索与嵌入那一层的直觉 + 指标」，不是「先考完研再写 RAG」。**

## 3. 该学哪些知识模块（按顺序）

1. **NLP 基础**：分词、编码、嵌入（embedding）是什么；余弦相似度。  
2. **信息检索**：倒排索引 vs 向量检索；混合检索（BM25 + vector）。  
3. **RAG 流水线**：chunk 策略、metadata、重排序（rerank，可选）、prompt 模板。  
4. **大模型使用**：API / 本地推理；温度、max tokens、system prompt；JSON 模式（若要做结构化输出）。  
5. **安全与合规（产品向）**：提示注入、数据最小化、审计日志；**不提供法律意见**的交互设计。  
6. **评测（轻量）**：准备 20～50 条「只有某文档里才有答案」的问题，测召回与忠实度。

## 6. 论文与报告（精读少而精）

> **与 §3.1 的关系**：**L2** 以 Lewis et al.（条目 1）对齐「你在写的 RAG 管线」；**L3 评测** 以 RAGAS / TruthfulQA（§6 评测段）对齐「怎么验收」；其余条目按 **§3.1 L4 行** 选读。

**RAG 主线（建议至少读摘要 + 方法图）：**

1. **Retrieval-Augmented Generation for Knowledge-Intensive NLP**（Lewis et al., 2020）— RAG 经典定义  
2. **In-Context Retrieval-Augmented Language Models**（相关后续工作选 1～2 篇综述即可）  
3. **Lost in the Middle**（Liu et al.）— 长上下文中间遗忘，**做 prompt 排布时要知情**  
4. **Self-RAG / CRAG**（了解「带自我校验的检索」概念即可，MVP 可先不做）

**评测与可信度：**

5. **TruthfulQA**、**RAGAS** 框架论文或文档 — 知道如何**自动粗评**「回答是否贴文档」

**法律 NLP（拓宽视野，非必读）：**

6. **LexGLUE / LegalBench** 类基准的**介绍页或短论文** — 了解领域任务类型，不强迫自己复现 SOTA

## 7. 工具链建议（学习与工程一致）

> **与 §3.1 的关系**：嵌入 / 向量库 / 编排 / 模型四列，分别支撑 **§3 模块 1、2～3、3、4**；上手顺序仍建议：**L1 先嵌入+向量库 → L2 再加编排与聊天模型**。

| 层级 | 工具 | 说明 |
|------|------|------|
| 嵌入 | `sentence-transformers`、OpenAI/Cohere 嵌入 API、`transformers.js` | MVP 可先 API，再换本地 |
| 向量库 | Chroma（本地）、LanceDB、SQLite-vss | 小数据够用 |
| 编排 | 先手写 50 行 RAG，再考虑 LangChain/LlamaIndex | 避免过早抽象 |
| 模型 | GPT-4o / Claude / 本地 Ollama + Qwen/Llama | 法律中文可优先试**通用强模型 + 好 prompt** |
