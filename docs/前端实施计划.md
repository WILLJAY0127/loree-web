## 阶段 0：横切基础（建议 1～2 天）

**目的**：减少后续各模块重复决策与返工。

1. **API 分层**：按 `features/<bc>/api/*.ts` 组织；列表筛选、分页参数与文档一致。
2. **TanStack Query**：为每类读模型定义稳定 `queryKey`；在单一模块（如 `shared/query/invalidation.ts`）或各 feature 内注释维护 **Command → 失效 keys** 矩阵，与《前端技术架构》缓存失效表对齐。
3. **反馈**：Toast（可先轻量实现）+ 删除/归档/驳回等 **二次确认**。
4. **三态**：Skeleton、空态插画+主操作、错误+重试（列表优先）。
5. **角色**：首屏角色选择（UI S1 全屏方案可迭代）；`RoleGuard` 与顶栏切换已具备，补「去设置切换」类引导即可。

## 阶段 1：M1 任务（建议 3～5 天）

**主要端点**（节选）：`GET /api/v1/tasks`、`GET /api/v1/tasks/{id}`、创建/编辑与各状态 `POST`/`PUT`（见 API 文档 M1）。

| 顺序 | 交付 |
|------|------|
| 1.1 | 任务列表 + 筛选 + 刷新 |
| 1.2 | 任务详情 + 按状态展示按钮 |
| 1.3 | 创建/编辑表单 + 校验 + 成功后失效 |
| 1.4 | 状态机操作按钮与错误处理 |
| 1.5 | 项目管理（与项目 API 同步） |

**阶段验收**：能走通「项目 → 任务 → 执行 → 提交 → 验收/驳回」；刷新后读模型一致。
