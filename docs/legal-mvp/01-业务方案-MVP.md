# 法律场景 MVP 业务方案（个人/小团队可落地）

> 定位：**非律师工具**，面向「有文书要整理、要检索、要对照模板」的普通用户与小企业；**数据默认在客户侧（本地或客户指定存储）**，降低信任成本。  
> 免责声明：本产品**不提供法律意见**；涉及诉讼、签约、合规决策，应咨询执业律师。

---

## 1. 为什么选这条

| 维度 | 说明 |
|------|------|
| 需求真实 | 合同、借条、劳动协议、租房、维权函等「有模板、有条款、要改要查」的重复劳动多 |
| 与「数据在客户侧」契合 | 文书内容敏感，本地/私有化存储是卖点，不是负担 |
| 无深专业门槛的切入口 | MVP 不做「判案」，做「整理、检索、比对、草稿辅助」 |

---

## 2. 目标用户（MVP 只做一类）

**首选：小微企业主 / 行政 / 法务助理（非持证律师）**

- 手头有多份 Word/PDF 合同版本，需要：**版本差异、关键条款摘录、到期提醒（手工录入日期）**  
- 需要按「自己的业务语料」检索：**以往签过的条款怎么写的**（数据全在本地）

暂不服务：持牌律所的冲突检索、法院专网、电子证据存证链（需资质与合作）。

---

## 3. MVP 范围（6～8 周可交付的形态）

### 3.1 必做（P0）

1. **本地文库**  
   - 导入文件夹：PDF / DOCX / TXT（MVP 可先 TXT + PDF 文本层）  
   - 元数据：文件名、导入时间、用户打的标签（如「供应商」「2024」）  
   - **所有全文索引与向量索引存本地**（浏览器 IndexedDB / 桌面端 SQLite，二选一技术路线）

2. **全文检索**  
   - 关键词搜索 + 高亮snippet  

3. **「条款卡片」摘录**  
   - 用户框选或粘贴一段文字 → 保存为卡片，附来源文档与页码/位置（若可解析）  

4. **对照模板（用户自备）**  
   - 用户上传「己方模板」与「对方发来的版本」→ **段落级或句子级 diff 展示**（规则 + 文本 diff，不依赖大模型也可先做）

5. **显式合规与产品声明**  
   - 首次使用勾选：「本工具不构成法律建议」  
   - AI 输出统一水印/提示：仅供参考  

### 3.2 选做（P1，有 AI 再做）

6. **问答式检索（RAG，全本地）**  
   - 仅基于用户导入的文档回答；**引用片段 + 文档名**  
7. **起草辅助**  
   - 从用户模板库填空生成初稿；若接模型，**禁止自动填充具体金额/管辖法院**等高风险字段，改为占位符  

### 3.3 不做（MVP 砍掉的）

- 自动判断「这份合同赢面」  
- 自动对接工商/裁判文书网实时抓取（合规与工程量大）  
- 多人协作实时编辑（可后续）

---

## 4. 数据在客户侧：架构原则

| 层级 | 要求 |
|------|------|
| 存储 | 文书原文、索引、向量、摘录卡片：**默认仅存用户设备或用户提供的目录** |
| 模型 | 优先：**本地小模型 / 端侧推理**；若用云端 API：**可选开关 + 脱敏提示 + 最小必要上传** |
| 日志 | 不上传文书内容到自有服务器；匿名用量统计可开关 |

---

## 5. 商业模式（学习用可先做免费）

- **Freemium**：本地检索 + diff 免费；云端大模型增强按次/订阅（可选）  
- **SMB 年费**：私有化部署包 + 更新（后期）

---

## 6. 风险与边界（写进 PRD）

- 幻觉导致错误条款 → **强制引用来源**、高风险段落人工确认  
- 隐私 → **默认本地**、导出加密备份  
- 监管 → 不以「律师」名义宣传；境内若涉及生成式服务备案，按届时规定执行  

---

## 7. 验收标准（MVP）

- [ ] 导入 ≥100 篇本地文档仍可检索（性能在可接受范围）  
- [ ] 任意检索结果可定位到原文位置或文件  
- [ ] 双版本 diff 可读  
- [ ] 无网环境下 P0 功能可用（若选本地模型路线）  

---

## 8. 与现有技术栈的映射（若用本仓库 React 做壳）

- **前端**：Vite + React + 本地存储适配层  
- **检索**：轻量可考虑 Orama / MiniSearch；向量路线需 `transformers.js` 或后端本地嵌入服务  
- **AI**：见同目录 `02-AI技术学习路径.md`
